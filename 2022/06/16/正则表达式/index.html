二、正则表达式底层实现

2.1 实例分析

  1. matcher.find()完成的任务：

    1. 根据制定的规则，定位满足规则的子字符串（比如1233）；
    2. 找到后，将子字符串的开始索引记录到matcher对象的属性int[] groups中，这里1233的1索引为0，groups[0] = 0, 将该子字符串的结束索引+1的值记录到groups[1] = 4；
    3. 同时记录oldLast的值为 子字符串的结束索引+1 即4；即下次匹配从oldLast开始

  2. matcher.group(0)分析：

    a. 根据groups[0]和groups[1]记录的位置，从content开始截取子串。

    b. 如果再次调用find, 则从下一处四个数字开始

  3. 分组时

    （1 记录整个字符串 ： groups[0] = 0; groups[1] = 4;

    （2 记录第一组： groups[2] = 0; groups[3] = 2;

    （3 记录第二组： groups[4] = 2; groups[5] = 4;

    （4 如果有更多的分组，继续…

    调用时：group(0):一整个匹配的字符；groups(1)：第一组；groups(2):第二组 ；

三、正则表达式语法

3.1 基本介绍

元字符从功能上分为：

  1. 限定符
  2. 选择匹配符
  3. 分组组合与反向引用符
  4. 特殊字符
  5. 字符匹配符
  6. 定位符

3.2 元字符-转义号\\

\\符号说明：检索特殊字符时，如& ，*，.，+ 或者$等符号，为了防止检索不到结果而产生报错，需要用到转义符，在Java语言中\\代表其他语言中\.

3.3 字符匹配符

符号 说明 示例 示例说明
[ ] 可接收的字符列表 [efgh] 匹配e,f,g,h中的任意一个
[^] 不接收的字符列表 [^abc] 匹配出除a,b,c任意字符之外的字符
- 连字符 A-Z 匹配大写A到Z的所有字符
. 匹配除\n之外的任意字符 a..b 匹配以a开头，以b结尾的长度为4的字符串，如adsb，a*&b
\\d 匹配单个数字字符，相当于[0-9] \\d 匹配0-9的单个数字
\\D 匹配单个非数字字符，相当于[^0-9] \\D 匹配除0-9之外的单个字符
\\w 匹配单个数字或者字母字符，相当于[0-9a-zA-Z] \\w{7} 匹配由数字和字母组成的长度为7的字符串
\\W 匹配单个除数字与字母之外的字符，相当于[^0-9a-zA-Z]
\\s 匹配任意空白字符，例如空格和tab
\\S 匹配任意非空白字符
` ` 选择匹配符 “han|韩|寒”

如何不区分大小写 – (?i): 例如， (?i)abc: 匹配abc且不区分大小写；a(?i)bc: b和c不区分大小写；a((?i)b)c:只有b不区分大小写；

或者Pattern pattern = Pattern.compile(regExp, Pattern.CASE_INSENSITIVE);

3.4 限定符

符号 含义 示例 说明
* 指定字符重复0次或n次 (abc)* 包含任意个abc的字符串，如abc，abcabcabc
+ 指定字符重复一次或n次 m+(abc)* 以至少一个m开头，后接任意个abc的字符串
？ 指定字符重复0次或1次 m+abc? 以至少一个m开头，后接ab或者abc
{n} 长度为n的字符 [abcd]{3} 由abcd中字符组成的任意长度为3的字符串
{n,} 指定至少出现n次
{m,n} 指定出现m-n次（闭区间）

3.5 定位符

符号 含义 示例 说明
^ 指定起始字符 ^[0-9]+[a-z]* 至少一个数字开头，后跟任意个小写字母
$ 指定结束字符 ^[0-9]\\-[a-z]+$ 以一个数字开头，后接-号，最后以至少一个小写英文字母结尾
\\b 匹配目标字符串的边界 han\\b 匹配以han结尾或者空格前有han的字符
\\B 非边界

3.6 分组

捕获分组

常用分组构造形式 说明
(pattern) 非命名捕获。捕获匹配的子字符串。编号为0的第一个捕获是由整个正则表达式模式匹配的文本，其他捕获的结果则根据左括号的顺序从1开始自动编号。
（?<name>pattern) 命名捕获。将匹配的子字符串捕获到一个组名称或编号名称中。用于name的字符串不能包含任何标点符号，并且不能以数字开头。可以使用单引号代替尖括号，例如(?’name’); name 就是组名

非捕获分组

构造形式 说明
(?:pattern) 匹配pattern但不捕获该匹配的子表达式，即它是一个非捕获匹配，不存储供以后使用的匹配，这对于”or“字符(|)组合模式部件的情况很有用，例如”industr(?:y|ies)”是比“industry|industries“更经济的表达式
(?=pattern) 非捕获匹配，例如”Windows (?=95|98|NT|2000)”匹配“Windows 2000”中的Windows但不匹配“Windows 10”中的Windows
(?!pattern) 该表达式匹配不处于匹配pattern的字符串的起始点的搜索字符串。它是一个非捕获匹配。例如，“Windows (?!95|98|NT|2000)”匹配”Windows 10”中的Windows但是不匹配”Windows 2000“中的Windows。

//d+?最后的问号表示非贪婪捕获。

四、应用实例

  1. 验证汉字：

    "^[\u0391-\uffe5]+$"

  2. 邮政编码：1-9开头的6位数

    "^[1-9]\\d{5}$"

  3. QQ号：1-9开头的5-10位数

    "^[1-9]\\d{4,9}$"

  4. 手机号码：13,14,15,18开头的11位数

    "^1[3458]\\d{9}$"

  5. URL:

"^((http|https)://)([\\w-]+\\.)+[\\w-]+(\\/[\\w=?/%-.&#]*)?"$

五、正则表达式三个常用类

5.1 Pattern类

pattern 对象是一个正则表达式对象。Pattern类没有公共的构造方法。要创建一个Pattern对象，调用其公共静态方法（compile），它返回一个Pattern对象。该方法接收一个正则表达式作为它的第一个参数。

5.2 Matcher类

Matcher对象是对输入字符串进行解释和匹配的引擎。与Pattern类一样，Matcher也没有公共的构造方法。需要调用Pattern对象的matcher方法来获取一个Matcher对象。

5.3 PatternSyntaxException

PatternSyntaxException表示一个非强制异常类，它表示一个正则表达式模式中的语法错误。

六、反向引用

  1. 分组

    用圆括号组成一个比较复杂的匹配模式，那么一个圆括号的部分我们可以看作是一个子表达式/一个分组

  2. 捕获

    吧正则表达式中子表达式/分组匹配的内容，保存到内存中以数字编号或显式命名的组里，方便后面引用，从左至右，以分组的左括号为标志，第一个出现的分组组号为1，第二个为2，以此类推。组0代表整个正则式。

  3. 反向引用

    圆括号的内容被捕获之后，可以在这个括号后使用，从而写出一个比较实用的匹配模式，这个我们称之为反向引用，这种引用既可以在正则表达式的内部，也可以是在正则表达式的外部，内部反向引用\\分组号,外部引用$分组号。
