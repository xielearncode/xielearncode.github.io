10.2 权限管理

ll或ls -l 第一部分为权限, 共十位

0 - 9位说明:

  1. 第0位确定文档类型(d, -, l, c, b):

    l 是链接, 相当于快捷方式

    d是目录, 相当于文件夹

    c是字符设备文件, 如鼠标, 键盘

    b是块设备, 如硬盘

    - 普通文件, 如reademe.txt

  2. 第1-3位确定该文件所有者拥有的权限**(r: read - 4 w: write - 2x: execute(可执行) - 1)**

  3. 第4-6位确定该文件所有组拥有的权限(rwx)

  4. 第7-9位确定其他组拥有该文件的权限

注意点:

​ 对某个文件有**写**的权限, 并不代表可以删除该文件或对其重命名, 只能对该文件所在目录有写的权限才可以删除该文件或重命名.

rwx作用到目录上时:

r 表示可查看目录内容

w 表示可对目录内文件进行创建, 删除, 重命名

x 表示可以进入该目录

权限后的数字含义: 文件: 直接是1, 目录: 子目录数+文件数

所有组后面的数字含义: 代表该文件大小 – 所占字节数

最后表示修改日期

修改权限

第一种方式: + , - , =

u: 所有者 g: 所有组 o: 其他人 a: 所有人(u, g, o的总和)

  1. chmod u=rwx, g=rx, o= x, 文件名/目录名 多个对象权限修改时使用逗号分隔
  2. chmod o+w 文件名/目录名
  3. chmod a-x

第二种方式: 通过数字变更权限 1, 2, 3, 4, 5 , 6, 7

r = 4, w = 2, x = 1

1: –x

2: -w-

3: -wx

4: r–

5: r-x

6: rw-

7: rwx

例:

​ chmod 751文件目录名 ==> chmod u=rwx,g=rx, o=x 文件目录名

修改文件所有者 - chown

chown newowner 文件/目录 : 改变所有者 : chown tom abc.txt

chown newowner:newgroup 文件/目录 改变所有者和所在组

chown -R newowner 文件/目录 如果是目录 则使其下所有子文件或目录递归生效

修改文件所在组 - chgrp

chgrp newgroup 文件/目录

chgrp -R newgroup 文件/目录

第十一章 定时任务调度

  • 概述

    任务调度: 是指系统在某个时间执行的特定的命令或程序

    任务调度分类: 1. 系统工作 2. 个别用户工作

10.1 crond任务调度机制

  • 基本语法

    crontab [选项]

  • 常用选项

    • -e: 编辑crontab定时任务
    • -l: 查询crontab任务
    • -r: 删除当前用户所有的crontab任务

  • 快速入门

    设置任务调度文件: /etc/crontab

    设置个人任务调度: 执行crontab -e 命令

    接着输入任务到调度文件, 如: */1**** ls -l /etc/ > /tmp/to.txt 表示每个小时的每分钟执行一次

  • 参数细节说明:

    项目 含义 范围
    第一个* 分钟 0-59
    第二个* 小时 0-23
    第三个* 几号 1-31
    第四个* 月份 1-12
    第五个* 星期 0-7(0和7都代表星期日)

  • 特殊符号说明:

    特殊符号 含义
    * 代表任何时间. 比如第一个*代表一小时每分钟都执行一次
    , 代表不连续的时间. 例如 0 8,12,16*** 就代表在每天的8点0分,12点0分, 16点0分都执行一次命令
    - 代表连续的时间范围. 比如 0 5 * * 1-6 代表周一到周六的凌晨五点0分执行命令
    */n 代表每隔多久执行一次, 比如*/10****代表每隔10分钟执行一次命令

  • crond相关命令

    crontab -e : 设定任务

    crontab -r : 终止任务调度

    crontab -l : 列出当前有哪些任务调度

    service crond restart : 重启任务调度

    注意: 写shell脚本后记得给它赋予执行的权限

10.2 at任务调度机制

  • 基本介绍

    • at命令是一次性定时计划任务, at的守护进程atd会以后台模式运行, 检查作业队列来运行
    • 默认情况下, atd守护进程每60秒检查作业队列, 有作业时, 会检查作业运行时间, 如果时间与当前时间匹配, 则运行此作业.
    • at命令是一次性定时计划任务, 执行完一个任务后不再执行此任务
    • 在使用at命令的时候, 一定要保证atd进程的启动, 可以使用相关指令来查看

    ps -ef | grep atd 检查atd是否在运行

  • at命令格式:

    at [选项] [时间]

    Ctrl + D 两次 结束at命令的输入

  • at 命令选项( 一般使用的频率较小)

选项 含义
-m 当指定的任务被完成后, 将给用户发送邮件, 即使没有标准输出
-I atq的别名
-d atrm的别名
-v 显示任务将被执行的时间
-c 打印任务的内容到标准输出
-V 显示版本信息
-q <队列> 使用指定的队列
-f <文件> 从指定文件读入任务而不是从标准输入读入
-t <时间参数> 以时间参数的形式提交要运行的任务
  • at指定时间的方式
    • hh:mm(小时:分钟). 加入该时间已经过去, 那么就放在第二天执行.
    • 使用midnight(深夜), noon(中午) , teatime(下午4点)等比较模糊的词语来指定时间
    • 采用12小时计时制, 即在时间之后加am或pm, 例如12pm指定具体日期, 指定格式为month day 或 mm/dd/yy 或dd.mm.yy, 指定日期必须跟在指定时间的后边. 例如 4:00 2021-03-01
    • 使用相对计时法. 指定格式为: now + count time-units now就是当前时间, time-units是时间单位, 这里可以是minutes, hours, days, weeks. count是时间数量. 例: now + 5 minutes
    • 直接使用today, tomorrow来指定完成任务的时间
  • 应用实例
    • 案例1: 2天后下午五点执行/bin/ls /home : at 17:00 + 2 days /bin/ls /home
    • 案例2: atq查看系统中没有执行的工作任务: atq
    • 案例3: 明天17点钟, 输出时间到指定文件内: at 5pm tomorrow date >> /home/mydate.txt
    • 案例4: 2分钟后, 输出时间到指定文件内: at now + 2 minutes date >> /home/mydate.txt
    • 案例5: 删除已经设置的任务: atrm [编号]

注意: 也可以直接执行脚本:

vim shname.sh 创建shell脚本文件

./shname.sh 执行shell脚本软件

第十二章 Linux磁盘分区|挂载

  • 原理介绍
    • 对Linux来说, 无论有几个分区, 分给哪一目录使用, 它归根结底只有一个目录, 一个独立且唯一的文件结构, Linux中每个分区都是用来组成整个文件系统的一部分.
    • Linux采用了一种叫”载入”的处理方法, 它的整个文件系统中包含了一整套的文件和目录, 且将一个分区和一个目录联系起来. 这时要载入的一个分区将使它的存储空间在一个目录下获得.
    • 示意图

12.1 挂载经典案例

通过案例来熟悉磁盘的相关指令和深入理解磁盘分区, 挂载, 卸载的概念

  • 如何增加一块磁盘

  1. 虚拟机添加硬盘: 点击虚拟机菜单设置==>选择硬盘==>点击添加==>点击下一步==>选择SCSI并点击两次下一步==>设置硬盘大小==>点击下一步完成==>点击确定后重启完成磁盘添加.

  2. 分区:

    分区命令: fdisk /dev/sdb(设备文件夹下的sdb文件夹–>硬盘sdb的映射)

    开始对sdb分区

    m 显示命令列表

    p 显示磁盘分区, 同 fdisk -l

    n 新增分区

    d 删除分区

    w 写入并退出

    说明: 开始分区后输入n, 新增分区, 然后选择p, 分区类型为主分区. 两次回车默认剩余全部空间. 最后输入w写入分区并退出, 若不保存退出输入q

  3. 格式化: mkfs -t ext4 /dev/sdb1 – (make file system) 其中ext4是分区类型-主分区

  4. 挂载(挂载后才能使用): 首先在/根目录创建一个/newdisk目录,然后挂载

    • cd /

    • mkdir newdisk

    • mount /dev/sdb1/newdisk /newdisk/ – 手动挂载方式mount [分区名称-绝对地址] [挂载目录]

    卸载使用: umount [分区名称-绝对地址]或[挂载目录], 然后可以重新挂载位置, 其中创建的文件不会丢失

    使用命令行挂载, 重启后会失效

  5. 设置可以自动挂载[永久挂载]

    永久挂载: 通过修改/etc/fstab实现挂载, 添加完成后执行mount -a即可生效

13.6 主机名解析过程分析

  • Host是什么

    一个文本文件, 用来记录IP和Hostname(主机名)的映射关系

  • DNS

    Domain Name System: 域名系统 . 是互联网上作为域名和IP地址相互映射的一个分布式数据库

  • 实例分析: 用户在浏览器上输入了 www.baidu.com

    1. 浏览器先检查浏览器缓存中有没有该域名解析IP地址, 有就先调用这个IP完成解析; 如果没有检查操作系统DNS解析器缓存, 如果有直接返回IP完成解析. 这两个缓存, 可以理解为本地解析器缓存.

    2. 一般来说, 当电脑第一次成功访问某一网站后, 在一定时间内, 浏览器或操作系统会缓存他的IP地址(DNS解析记录). 在cmd窗口输入

      ipconfig /displaynds : DNS域名解析缓存

      ipconfig /flushdns : 手动清理dns缓存

    3. 如果本地解析器缓存没有找到对应的映射, 检查系统中hosts文件中有没有配置对应的域名IP映射, 如果有, 则完成解析并返回.

    4. 如果本地DNS解析器缓存 和 hosts文件中都没有对应的IP, 则到域名服务DNS进行解析域

    浏览器缓存(浏览器) ==> DNS解析器缓存(操作系统) ==>hosts文件(文件)==> 域名服务DNS(服务器) ==>域名不存在

    注意: ping的话就经过浏览器缓存

第十四章 进程管理

14.1 基本介绍

  1. 在Linux中, 每个执行的程序都称为一个进程. 每一个进程都分配一个ID号. (pid, 进程号)
  2. 每个进程都可能以两种方式存在着. 前台和后台, 所谓前台进程就是屏幕上可以操作的, 后台就是屏幕没有显示, 但是在运行
  3. 一般系统的服务都是以后台的方式运行的, 且常驻在系统中, 直到关机才结束

14.2 显示系统执行的程序

PID : 进程识别号

TTY : 终端机号

TIME : 此进程所消耗CPU时间

CMD : 正在执行的命令或进程名

VSZ : 进程占用虚拟内存的大小

RSS : 进程占用物理内存大小

STAT : 进程状态，其中S-睡眠，s-表示该进程是会话的先导进程，N-表示进程拥有比普通优先级更低的优先级，R-正在运行，D-短期等待，Z-僵死进程，T-被跟踪或者被停止等等

STARTED : 进程的启动时间

  • 基本介绍

    ps命令是用来查看目前系统中, 有哪些正在执行, 以及他们执行的情况. 可以不加任何参数

    • ps -a : 显示当前终端的所有进程信息

    • ps -u : 以用户的格式显示进程信息

    • ps - x : 显示后台进程运行的参数

    • 一般上述三个组合使用 ps -aux

  • 应用实例: 以全格式显示当前的进程, 查看进程的父进程

    • ps -ef : 以全格式显示当前进程, e所有进程, f全格式

    • BSD风格

      • C : 优先级, 越小越先
      • PPID : 父进程PID
      • STIME : 进程启动的时间

14.3 终止进程kill 和 killall

  • 介绍

    若是某个进程执行到一半时就需要停止时, 或是已消耗了很大的系统资源时, 此时可以考虑停止该进程. 使用kill命令来完成此项任务.

  • 基本语法

    • kill [选项] [进程PID]
    • killall [进程名称] : 子进程会被干掉

  • 常用选项

    • -9 : 表示强迫进程立即停止

  • 实践案例

    • 终止远程登录进程: kill [进程sshd -D编号] ==> 恢复: /bin/systemctl start sshd.service
    • 终止多个gedit (文本编辑器): kill gedit
    • 终止终端: kill -9 11320

14.5 服务管理

  • 介绍

    服务本质就是进程, 但是是运行在后台的, 通常都会监听某个端口, 等待其他程序的请求, 比如mysql, sshd, 防火墙. 因此我们又称之为守护进程.

  • service管理指令

    • service [服务名] [start | stop | restart | reload | status]
    • 在CentOS7.0之后, 很多服务不再使用service, 而是使用systemctl
    • service指令管理的服务在/etc/init.d查看

  • service管理指令案例

    • 使用service指令, 查看, 关闭, 启动 network : service network stop service network start

  • 查看服务名

    • 方式一: 使用setup ==> 系统服务就可以看见全部
    • 方式二: /etc/init.d/ 看到service指令管理的服务 ls -l /etc/init.d

  • 服务运行级别runlevel

    1. 运行级别0 : 系统停机状态, 系统默认运行级别不能设置为0, 否则不能正常启动
    2. 运行级别1 : 单用户工作状态, root权限, 用于系统维护, 禁止远程登录
    3. 运行级别2 : 多用户状态(没有NFS), 不支持网络
    4. 运行级别3 : 完全的多用户状态(有NFS), 登录后进入控制台命令行模式
    5. 运行级别4 : 系统未使用, 保留
    6. 运行级别5 : X11控制台, 登陆后进入图形GUI模式
    7. 运行级别6 : 系统正常关闭并重启, 默认运行级别不能设置为6, 否则不能正常工作.

  • 开启启动流程说明:

    • ①开机 ==> ②BIOS ==> ③/boot ==> ④systemd进程1 ==> ⑤运行级别 ==> ⑥运行级别对应的服务

  • CentOS7后运行级别说明

    • 在/etc/initab

      可以简化命令: multi-user.target : analogous to runlevel3

      ​ graphical.target : analogous to runlevel 5

    • 查看当前运行级别: systemctl get-default

    • 设置运行默认运行级别: systemctl set-default TARGET.target

  • chkconfig指令

    • 介绍
      • 通过chkconfig命令可以给服务的各个运行级别设置自动启动或关闭
      • chkconfig指令管理的服务在/etc/init.d查看
      • 注意: 在CentOS7以后, 很多服务用systemctl管理
    • 基本语法
      • 查看服务 : chkconfig --list [| grep xxx]
      • chkconfig 服务名 --list
      • 自启动/自关闭 : chkconfig --level 5 服务名 on/off

  • systemctl指令

    • 基本语法 : `systemctl [start | stop | restart | status] 服务名
    • systemctl指令管理的服务在/usr/lib/systemd/system查看
    • systemctl设置服务自启动状态
      • systemctl list-unit-files [| grep xxx] `查看服务开机启动状态
      • systemctl enable 服务名 设置服务开机启动
      • systemctl disable 服务名关闭服务开机启动
      • systemctl is-enabled 服务名查询某个服务是否是自启动
    • 细节讨论
      • 关闭或启动防火墙后, 立即生效(用telnet测试)
      • 指令临时生效, 重启后恢复. 要永久生效需使用enable或者disable代替stop和start

  • firewall指令 =================== (netstat -anp | more 查看端口或协议)

    • 打开端口: firewall-cmd --permanent --add-port=端口号/协议
    • 关闭端口: firewall-cmd --permanent --remove-port=端口号/协议
    • 重新载入, 才能生效: firewall-cmd --reload
    • 查询端口是否开放: firewall-cmd --query-port=端口/协议

14.6 动态监控进程

  • 介绍

    top与ps命令很相似, 他们都用来显示正在执行的进程, top与ps最大的区别就是top执行时间内可以更新运行的进程(即动态监控)

  • 基本语法

    top [选项]

  • 选项说明

    • -d 秒数 : 指定top每隔几秒跟新, 默认3秒
    • -i : 让top不显示闲置或者僵死进程
    • -p 进程ID : 通过指定监控进程id来仅仅监控某个进程的状态

  • 交互操作说明(查到后的操作):

    • P : 按照cpu使用率进行排序, 默认此项
    • M : 以内存的使用率排序
    • N : 以PID排序
    • q : 退出
    • u : 监控特定用户
    • k : 通过PID终止某个进程

  • 监控网络状态

    • 基本语法 : netstat [选项]
    • 选项说明:
      • -an : 按一定顺序排列输出
      • -p : 显示哪个进程在调用

  • 检测主机连接的命令: ping 对方ip地址

    • 作为一种网络检测工具, 它主要是用检测远程主机是否正常, 或是两部主机间的网线或网卡故障
